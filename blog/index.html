三模块

MLIRGen

遍历抽象语法树

Dialect

负责定义各种操作和分析,具备可拓展性

TableGen

使用Operation Definition Specification (ODS)框架进行自动化生成代码，整个框架是基于TableGen规则来完成相应功能,TableGen定义各种类

模块组装

这图还挺清晰的

TableGen

是一种声明性编程语言(Declarative Programming Language)

开发者定义目标与要实现的结果,由语言的执行引擎具体实现

简洁,高抽象,可读性强,常见的有SQL,CSS…

TableGen描述硬件架构,指令集,寄存器,操作符等属性,并以此自动生成目标代码,源代码中以.td文件形式存在,编译时自动生成C++相应文件,为Dialect模块文件提供支持

以Toy语言为例,深入研究TableGen

1. Dialect链接

首先我们在.td文件中定义一个TableGen与Dialect的链接,负责把Dialect中定义的所有OPeration整合

2. 创建Operation基类

构造所有Dialect OPeration 的基类Toy_Op,所有的Operation都会基于此类构造

3. 自定义Operation

所有的类都继承上述基类,以TransposeOp为例,使用TableGen规则定义参数,值,builder,verifier等

而一个operation需要指定:

  • operation名字:在MLIR中唯一的标识符
  • operation的特性:下文会提到,例如是否要移除side-effect,是否要验证操作数和结果类型是否相同,由C++特性提供支持
  • 操作的参数,操作的结果
  • 操作的描述和方言的特定信息

4. TableGen生成C++代码

利用MLIR中的mlir-tblgen工具进行编译,编译时TableGen会把.td文件生成为C++文件,生成的代码会为DIalect提供支持

Operation Definition Specification(ODS)

一种Operation定义的规范, 按这种规范写的自定义Operation,可以拓展为等价的mlir:Op的C++代码.

MLIR Rewrite(重写)的用途

  • 优化(Optimization): 常量折叠,消除冗余操作
  • 降级(lowering):高级Dialect->低级Dialect
  • 规范化(Canonicalization):同一IR形式

使用上述两种方法确实可以对表达式进行自定义变形,但是只能针对特定的一个Operation变型,而相似的变型却不能直接套用,导致代码重复,徒增工作量

我们引入interfaces(接口)机制,针对常见的优化模式,编写可复用的变型代码

在PassManager中添加的Pass会按照指定顺序进行执行，MLIR框架会自动调用这些Pass的runOnOperation方法，在这些方法中执行具体的优化任务

这里提到了内联(inline) pass和Shape推断pass

内联Pass

内联Pass可以用定义的函数体代替函数的调用. 对于一些小的函数,对其调用,返回的准备和收尾工作或许比本身代码大得多,因此对这类简单的,使用频繁的小函数,声明为内联函数能提高效率

Dialect接口:

MLIR本身提供了DialectlinerInterface接口,我们只需要继承它然后实现所需函数,就能自定义内联操作

Operation接口:

我们还需要知道函数在哪里内联,MLIR提供了CallOpInterface接口,会对函数调用进行标记,告知内联操作 函数调用的位置

而在函数调用时的类型与函数定义的类型可能不一致,在定义时参数是泛化的tensor<*xf64>,任意的,而调用时候参数是确定的类型tensor<2x3xf64>,

MLIR 的类型系统是静态的、严格的，函数定义和函数调用处的类型必须完全匹配才能合法内联，否则 verifier 会直接报错(在上面的Operation特性有提到过) 然后我们也不能把泛化累心转化为特定类型,可能会发生错误,因为这是把大范围变成小范围,可能不在小范围内,是未定义行为

再次我们添加CastOp,用来将确定的类型转变为泛化的类型,我们在Operation中顶i一CastOp,然后再Dialect模块的内联操作接口中重写相应函数,从而再MLIR表达式中加入CastOp

部分Lowering

源程序从IR到运行再处理器上面,还要进行一系列lowering,部分lowering指的是 从一个高级别dialect到低级别dialect的过程中,只lowering一部分operation,剩下的operation只需要经过升级能够与其他DIalect共存即可

我们要使用MLIR提供的仿射变换Dialect:Affine,但是该Dialect不支持 toy中的print,所以我们对print操作进行升级

此时,print可以同时接受F64Tensor和F64MemRef类型的参数,后者是affine dialect支持的类型,实现共存

ToyToAffine模块

将原有的MLIR表达式lowering到新表达式,就是把原来的Operation换成更接近底层的Operation,所以我们在ToyToAffine模块中,要把原来的TToyOperation识别出来,用Affine dialect里的operation进行替换,与此同时在写pass时添加新dialect 禁用原来的toy dialect,这样就完成了Lowering

首先,在ToyToAffine模块中,针对每个要替换的Operation定义匹配和重写规则

然后,编写pass,使用addLegalDialect函数把Affine和StandardDialect进行合法化,并禁用Toy Dialect,但是前者不具备print op,所以在pass中还要print的支持,所以把printOp加到Dialect中,实现部分Lowering

此时表达式还有冗余部分,即转置相乘分在了两个循环内进行,此时使用LoopFusionPass和MemRefDataFlowPass消除冗余

printIndent()

完整流程:

CanonicalizerPass.cpp阅读

发表于 2026-03-08 更新于 2026-04-20 分类于 MLIR

Canonicalizer.cpp

这个pass其实很常用了,平时跑mlir-opt -canonicalize基本就在用它,所以还是值得单独看一下

整体结构

目的:

这个pass本质上不是自己写很多特殊优化逻辑,而是统一收集各个Dialect和Op上声明好的canonicalization patterns 然后用贪心重写框架去反复应用

也就是说

  • 具体怎么折叠
  • 哪些op可以简化
  • 哪些组合可以合并

这些规则大部分不是写在CanonicalizerPass里,而是分散在各个op定义和对应实现里

而这个pass做的事情更像是一个统一调度器

Pass定义

Canonicalizer继承自PassWrapper<Canonicalizer, OperationPass<func::FuncOp>>这种印象其实不对

它这里是更泛化的写法,可以对任意隔离的操作运行,本质上还是一个OperationPass

核心点是: canonicalize不是只服务某一个Dialect,而是一个全局通用整理流程

命令行参数

所以执行的时候就是

核心逻辑

runOnOperation()

这个函数里面核心流程其实比较清楚

  1. 取当前操作 getOperation()
  2. 创建RewritePatternSet
  3. 从当前操作以及嵌套操作相关的Dialect / Op里收集canonicalization pattern
  4. 调用applyPatternsGreedily()反复应用

所以这个pass核心并不在于自己怎么改IR,而在于把大家已经定义好的规则组织起来

Pattern的收集

这里值得注意的是 它不是手写一个个patterns.add<...>()

而是通过MLIR里已经提供好的接口去收集

也就是如果某个Op实现了

那么canonicalizer pass在跑的时候就能把它拿到

这也是为什么平时在自己写Dialect或者写Op的时候,经常会看到:

  • hasCanonicalizer
  • hasCanonicalizeMethod
  • fold()

这些东西最后都能和canonicalize这条链路接起来

为什么常配合fold

canonicalize和fold经常一起提

因为很多最简单的简化:

  • 常量折叠
  • 冗余维度消除
  • 无意义转换删除

这些是可以直接在fold()里做掉的

而更复杂一点,需要匹配多条op组合关系的,就更适合放进canonicalization pattern

所以可以理解成:

  • fold() 更像局部立即折叠
  • canonicalization pattern 更像重写规则

两者最后都服务于把IR整理得更干净

为什么要用贪心重写

这里底层调用的是applyPatternsGreedily()

原因也比较自然,因为一个pattern生效后,往往会创造出新的可优化机会

例如:

先把opB折掉以后,可能opA和opC之间又出现新的匹配机会

所以这里不是跑一遍就结束,而是不断迭代直到收敛

这个Pass本身值得学什么

我觉得看这个文件主要是学三件事

第一: canonicalize到底是什么

它不是某个单独优化技巧,而是一套统一整理IR的入口

第二: pattern和pass的关系

以前会容易觉得“我写了pattern就是写了pass”

其实不是

  • pattern定义怎么改
  • pass决定什么时候改,在哪个范围改,用什么策略改

canonicalizer就是一个很标准的“pattern驱动型pass”

第三: 为什么自己写pass时经常先接canonicalize

因为很多自己新建出来的op,或者替换后的中间结果,不一定是最简形式

这时候跑一次canonicalize

  • 可以清理中间垃圾
  • 可以触发已有pattern
  • 让IR更稳定

这也是为什么很多pipeline里都喜欢看到

和自己写Pass的关系

如果后面自己写Buddy或者MLIR的小优化,其实有两个方向

直接自己在pass里改

比如手动遍历op,手动replace,手动erase

这种写法适合很直接的逻辑

把规则写成pattern,再交给贪心驱动

这种方式更自然,也更贴合MLIR整体风格

而CanonicalizerPass.cpp就是这种思想的典型入口

所以这个文件虽然本身不长,但是位置很关键

它把“各个地方零散定义的canonicalization规则”真正串起来了

分类: 分类任务结果种类是固定的,比如说 输出结果是猪狗猫,模型会告诉你是这三个中的一种,不可能是牛

w权重,b偏置

误差(输出与真实值之间的差距)计算 设定误差函数

y =wx+b y:特征 w: 标签

  1. 穷举法

  2. 最小二乘法,

  3. 梯度下降法 学习率(a), 三个中最优算法

数据是否带标签然后输入到模型训练是有无监督学习的区别,有监督学习带标签

逻辑回归模型(分类任务，预测类别概率)

逻辑回归算法是二分类, 可以理解为利用回归的方法进行分类,

sigmoid函数:
$$
\sigma(x)=z = \frac{1}{1 + e^{-x}}
$$
导数:
$$
\sigma’(x) = \sigma(x) \left(1 - \sigma(x)\right)
$$

我们将x转化为输入的数据特征,这个时候可以利用sigmoid函数对其进行映射,

z=w0x0+w1x1+…+wnxn + b

此时根据z值 靠近1或0 得到二分类结果,

评价指标:

分类模型评价指标: 准确率,精确率,召回率,F1值

回归模型评价指标: 平均绝对误差,均方误差(loss值),均方根误差,MAPE(平均绝对百分比误差) 都大差不差

深度学习入门

全连接神经网络(fully connected neural network,FCNN或MLP:multilayer perception)

最基础的神经网络结构,CNN,RNN,transformer可以看作其进化版

整体结构:
输入层-> 隐藏层->输出层

神经元的数学表达式:

$$
a = h(wx + b)
$$

h是激活函数,激活函数让神经网络从线性变成非线性,打破每一层之间的线性关系,赋予网络非线性能力

例如前面所学的sigmoid函数就是一个激活函数

每个神经元对应一个b,每个输入的x 对应一个参数w,

所以这一层参数个数就是(x的数量乘上神经元数量) 再加上神经元的数量

激活函数

sigmoid优点: 简单 非常适用分类任务

Tanh函数:
$$
\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}
$$
导数

$$
\tanh’(z) = 1 - \tanh^2(z)
$$
ReLU函数:
$$
f(x) = \begin{cases} 0 & \text{if } x > 0 \ x & \text{if } x \le 0 \end{cases}
$$

$$
\frac{d}{dx}\operatorname{ReLU}(x)=\begin{cases}1,&x>0\0,&x\le0\end{cases}
$$

优点: 解决了梯度消失(>0情况下) ,计算更为简单

缺点: 训练时可能出现神经元死亡(x<0,导数为0,w和b不更新了)

Leaky ReLU函数:
$$
f(x) = \begin{cases} 0 & \text{if } x > 0 \ ax & \text{if } x \le 0 \end{cases}
$$
解决了神经元死亡问题

前向传播

从输入开始，经过一层层加权计算、非线性变换，最终得到输出结果

也就是y=model(x) ,输入x得到y的过程

神经网络的损失函数 loss

f(x) 就是关于w,b的函数嘛

反向传播

案例

利用损失函数,通过链式法则，从输出层向前一层层计算每个参数的梯度，使得我们能知道如何更新参数以降低 loss。

梯度下降法是反向传播具体的一个实现过程 利用损失函数的导数来更新w,b

所以前面导数为0,就会出现神经元死亡,w,b都不更新了怎么整

梯度下降法参数更新公式:

a就是学习率,根据其大小,调整w跨度,大了可能错失最小w,小了时间资源花费大

本身w的导数就是一个二次函数,其存在最小值 我们的目的就是尽可能地逐步更新w 然后找到他,这样 损失值就越来越小,与真实值差距就减少了,模型是不是就更准确了啊

b同理

当然这里为什么是减去a*导数,因为导数是正的话,向右走,我们得减去他才能往山沟里(最低点)走啊,如果导数是负的,那就在中心线左边啊,那就是往左边走了,减去他才往右走

过拟合:

模型的复杂度太高（参数太多或结构太灵活），导致它在训练数据上学习到了随机误差，而不是数据真正的统计规律。

常见解决办法包括：

  • 减少模型复杂度（比如浅一点的神经网络、少一点参数）
  • 正则化（Regularization），对模型参数加约束
  • 数据增强或增加训练样本
  • 使用交叉验证选择合适的超参数
