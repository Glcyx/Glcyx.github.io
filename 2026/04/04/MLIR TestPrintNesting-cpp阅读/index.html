TestPrintNesting-cpp阅读

发表于 2026-04-04 更新于 2026-04-05 分类于 MLIR

TestPrintNesting.cpp

整体结构

目的:

这个Pass用来遍历和打印IR的嵌套结构(operation->region->block->operation..)

Pass定义:

TestPrintNestingPass继承自PassWrapper<TestPrintNestingPass, OperationPass<>>

方法实现:

核心逻辑在runOnOperation()中,通过递归遍历IR结构

Pass编写规范

继承结构

PassWrapper<YourPass,BasePass> ,BasePass可以是OperationPass<>(操作级),ModulePss<>(模块级),等等..

这里用前者 表示pass从一个操作开始运行,通常是模块的根目录

方法解析

MLIR_DEFINE_EXPLICIT_INTERNAL_INLINE_TYPE_ID(TestPrintNestingPass)定义pass类型的ID,用于内部识别. 这是MLIR的宏,必须添加

void runOnOperation() overridepass核心逻辑的入口,override表示重写基类方法,继承不同的积累,重写对应的方法

辅助方法:

如 printOperation、printRegion、printBlock，用于递归处理IR。

IdentRAII :管理缩进,保证作用域结束后自动减少缩进

通过indent来模拟IR的嵌套结构,嵌套越深indent越大,该层嵌套结束,通过析构函数自动减少indent大小

int indent:

在runOperation()被调用时初始化为0

一个indent代表两个空格

RAII类struct IdentRAII
IdentRAII pushIndent()

++indent修改了pass对象的indent成员数值

引用绑定机制: IdentRAII(++indent)在构造对象的时候,参数int &indent绑定到pass的indent(因为++indent返回的是左值引用

最后返回IndentRAII对象,其成员indent引用pass成员indent

printIndent()

llvm::raw_ostream &printIndent() 输出缩进空格，返回 llvm::outs() 引用

链式调用:llvm:raw_ostream实现了对<<的重载,每次<<都返回同一个流引用,可以无限链式调用

完整流程:

  • llvm::outs()返回流引用,类型是llvm::raw_fd_ostream& 但用基类llvm::raw_ostream& 接收,实现多态设计
  • printIndent() 返回这个引用。
  • << "text" 调用基类的 operator<<，输出数据，并返回流自身的引用（return *this;）。
  • 链式调用继续操作同一个对象。

实现逻辑

通过递归调用printOperation()->printRegion()->printBlock()->printOperation()实现遍历

通过pushIndent()和printIndent()来管理输出格式,使得嵌套结构更加清晰

使用llvm.out()输出到标准输出

在MLIR命名空间内定义void registerTestPrintNestingPass()函数 使用PassRegistration<TestPrintNestingPass>()来注册 允许该pass被动态加载或在命令行中被MLIR框架使用

执行流程

Pass通过PassManager管理,通常使用命令: mlir-opt --test-print-nesting xxx.mlir执行

通过resetIndent()初始化indent为0

打印根操作: 调用printOperation(op)输出操作的基本信息

递归进入Region: 对于每个Region,调用printRegion(region)输出块数量

递归进入Block: 对于每个Block,打印参数数量,后继者,操作数量

递归调用Operation:每个块递归调用Operation,不断循环

每次进入新的嵌套层级,调用pushIndent(),增加缩进,离开时,因里开作用域,自动调用析构函数,减少indent大小
